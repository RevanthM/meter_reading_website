# AMR Portal — User Manual

**Meter Reading Operations Portal**  
Review field and simulator captures, correct readings, route sessions to training or unit tests, and manage model-training pipelines.

| | |
|---|---|
| **Audience** | Reviewers, test-data reviewers, labelers, admins |
| **Browser** | Chrome or Edge (latest recommended) |
| **Screenshots** | [user-manual/images/](user-manual/images/) |

---

## 1. Signing in

1. Open the portal URL provided by your administrator.
2. Sign in with your **Anica user ID and password** (device verification / OTP if your account requires it).
3. After login you land on the **Dashboard** with the sidebar on the left.

![Login screen](user-manual/images/01-login.png)

**Access denied?** Contact your IT administrator.

---

## 2. Portal layout and roles

### Main areas

| Area | Purpose |
|------|---------|
| **Sidebar** | Navigation, role selector, work type |
| **Header** | Page title, filters, actions |
| **Content** | Lists, charts, session detail |

![Sidebar — reviewer role](user-manual/images/03-sidebar-reviewer.png)

### Role selector

Use the **Role** dropdown at the top of the sidebar. The role controls which menu items appear and what you are allowed to save.

| Role | Typical user | What you can do |
|------|----------------|-----------------|
| **reviewer** | Field QA / operations | Review new captures, set correct/incorrect, fix readings, send to training or test dataset, set image difficulty |
| **test data reviewer** | ML / QA | Approve test-dataset picks into unit test images; manage unit test manifest |
| **labeler** | Training ops | Move sessions through training pipeline folders; Model Training Center |
| **admin** | Platform owner | Everything labeler has, plus Model Factory, iteration registry, all readings, manual upload |

The role is stored in your browser. Pick the role that matches the task you are doing.

### Welcome dialog

On first visit you may see a **Welcome** dialog with a status glossary. Close it for this session, dismiss it permanently, or reopen it from the sidebar **Help** control.

![Welcome dialog](user-manual/images/02-welcome-modal.png)

## 3. Work type and data source

### Work type

Meter data is grouped by **work type** (for example `1000`, `2000`, … `5000`). Use the work-type control in the sidebar to switch buckets. All lists and counts apply to the selected work type only.

### Field vs simulator

Sessions are tagged as **Field** (real captures) or **Simulator** (synthetic). This affects folder names when sessions are promoted to “correct” storage. Manual uploads let you pick source at upload time.

## 5. Reviewer workflow

### 5.1 Awaiting review

**Sidebar → Awaiting review**

Lists captures that are **not manually reviewed** yet.

![Awaiting review list](user-manual/images/04-awaiting-review-list.png)

**Typical steps:**

1. Open a session from the list.
2. Inspect images and model reading.
3. Set outcome: **Correct**, **Incorrect**, **No dials**, or **Not sure**.
4. Enter or correct the **4-digit reading** if needed.
5. Optionally set **image difficulty** (Normal / Difficult / Very difficult).
6. Optionally mark **Send to training dataset** or **Send to test dataset**.
7. Click **Save**.

### 5.2 Wrong (reviewed)

Sessions already reviewed as incorrect appear in pipeline folders (analyzed → labeled → training). Use **Wrong (reviewed)** in the sidebar.

![Wrong (reviewed) queues](user-manual/images/11-wrong-queues.png)

### 5.3 Correct

**Sidebar → Correct** — sessions accepted as good reads after review.

![Correct list](user-manual/images/10-correct-list.png)

### 5.4 List filters

Most list pages support date presets, search by session ID or reading, and cohort filters where applicable. Click a row to open **session detail**.

## 6. Manual bulk upload and labeling

Available for **reviewer**, **test data reviewer**, and **admin**.

### 6.1 Bulk upload

**Sidebar → Bulk upload**

Upload many meter photos at once. Label readings on the next screen.

![Bulk upload](user-manual/images/06-bulk-upload.png)

| Step | Action |
|------|--------|
| 1 | Click the drop zone or **Choose files**, or drag JPEG/PNG images onto the zone |
| 2 | Review thumbnails; use **Clear selection** to start over |
| 3 | Choose **Source**: **Simulator** or **Field** |
| 4 | Click **Upload N images** |
| 5 | Open **View uploaded images** to label |

![Bulk upload with files selected](user-manual/images/07-bulk-upload-with-files.png)

**Notes:** Drag-and-drop **adds** files; the file picker **replaces** the selection. Batch size is limited by the server (typically 50 images).

### 6.2 Label uploads

**Sidebar → Label uploads**

| Filter | Shows |
|--------|--------|
| **New** | Uploads still needing a 4-digit reading |
| **Reviewed** | Labeled uploads |
| **All** | Everything matching the date filter |

![Label uploads grid](user-manual/images/08-label-uploads.png)

**To label:**

1. Click a thumbnail (opens **lightbox** with zoom and pan).
2. Type a **4-digit reading** (leading zeros allowed, e.g. `0047`).
3. Click **Save label** or **Save & next**.

![Label lightbox](user-manual/images/09-label-lightbox.png)

## 7. Session detail (reading detail)

Open any session from a list to inspect images, readings, and metadata.

![Session detail](user-manual/images/05-reading-detail.png)

| Control | Effect |
|---------|--------|
| **Status** | Moves session to the matching workflow folder |
| **User correction / ML prediction** | Updates stored metadata |
| **Dial digits** | Per-dial edits when dial crops exist |
| **Training / test dataset** | Reviewer routing flags |
| **Prev / Next** | Queue navigation when opened from a list |

In **labeler** role, images are read-only here; use pipeline pages to move stages and **reviewer** role for outcomes.

## 12. Troubleshooting

| Problem | What to try |
|---------|-------------|
| Blank lists after login | Confirm **work type** in sidebar; wait for loading to finish |
| Save fails with 403 | Switch to **reviewer** role (admin uses reviewer save permissions) |
| Upload fails | Check file type (JPEG/PNG), size limits, and network |
| Old UI after deploy | Hard refresh (Ctrl+Shift+R / Cmd+Shift+R) |
| Model Factory empty | Server must have Roboflow API key configured |

## Appendix A — Status glossary

| Status | Meaning |
|--------|---------|
| **Correct** | Reviewed good read |
| **Awaiting review** | Not manually reviewed yet |
| **Analyzed** | Incorrect pipeline — diagnostics reviewed |
| **Labeled** | Ground truth applied for training |
| **Training** | In or ready for training dataset |
| **No dials** | No usable dials in image |
| **Not sure** | Ambiguous — needs human decision |
| **Manual upload** | Portal bulk upload queue |
